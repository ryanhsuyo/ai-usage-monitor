# Product Brief

## 定位

本機優先的 AI 訂閱**用量監控、額度預測、重置提醒與方案決策**桌面工具。不是單純顯示 token 數字，而是幫使用者回答：

1. 目前的 AI 訂閱方案是否划算？
2. 照目前速度，本週額度可能什麼時候用完？
3. 重置之前，預估還能完成幾次相似任務？
4. 下一次額度什麼時候重置？
5. 重置（預計/確認）時能否主動通知？
6. 能否透過桌面、Discord、Slack、Telegram、自訂 Webhook 收通知？
7. 哪些任務、模型或專案最消耗額度？
8. 應該升級、維持還是降級？

## 目標使用者

重度 AI 訂閱使用者（Claude Pro/Max、ChatGPT Plus、Codex、Cursor…），常在週期中段擔心額度不夠、或懷疑自己付太多錢的人。

## 差異化

- **本機優先**：無雲端、無帳號、資料不外流；一個 .dmg 拖進 Applications 就能用
- **誠實的估算**：所有預測附 0–1 可信度與原因；資料不足直接說，不亂編數字
- **決策導向**：輸出是「還能做 N–M 次 coding」「建議維持方案」這種可行動結論
- **可擴充**：Provider / 通知管道 / 平台能力全部走 Adapter，第一版 Claude，架構天生支援 Codex/ChatGPT/Gemini/Cursor

## 明確不做（第一版）

雲端後端、會員系統、Supabase、爬蟲式自動同步（Roadmap Phase 3 前不做）、假資料冒充真實 Provider 數據、精確 token 計數（官方未公開）。
